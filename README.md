# Demand Planning ARM (Automated Risk Management)

Прототип аналитической системы поддержки принятия решений для прогнозирования спроса в розничной торговле. Решение автоматизирует полный цикл планирования: от загрузки и очистки данных до генерации заказов и управления исключениями. Разработано в рамках курса «Информационные системы прогнозирования и планирования цепи поставок».

## Функциональные возможности

Система реализует гибридный подход к прогнозированию и иерархическому согласованию данных.

*   **ETL и оптимизация данных**
    Обработка сырых данных датасета M5 (Walmart). Приведение типов (Downcasting) и оптимизация памяти для работы с большими массивами. Генерация календарных и ценовых признаков.

*   **Сегментация спроса**
    Автоматическая классификация временных рядов на основе метрик ADI (Average Demand Interval) и CV² (Coefficient of Variation). Товары разделяются на кластеры: Стабильный, Эпизодический, Вариативный, Спорадический.

*   **Движок прогнозирования (Forecasting Engine)**
    Гибридная архитектура на базе экосистемы Nixtla.
    *   **Статистический подход:** AutoARIMA с аппроксимацией для стабильных рядов.
    *   **Машинное обучение:** XGBoost с генерацией лагов и скользящих окон (MLForecast) для сложных нелинейных зависимостей.

*   **Иерархическое согласование**
    *   **Bottom-Up:** Агрегация прогнозов уровня SKU до магазина/категории.
    *   **Top-Down:** Дезагрегация прогноза категории на основе исторических долей (Average Historical Proportions) для сглаживания шума.

*   **Управление рисками (ARM)**
    Модуль Exception Management. Автоматическое выявление аномалий по правилу трех сигм. Генерация прескриптивных рекомендаций (Увеличить/Снизить закупку). Интерактивная корректировка причин отклонений.

## Технологический стек

Проект реализован на языке Python 3.10+ с использованием современных библиотек анализа временных рядов.

*   **UI/Web:** Streamlit
*   **Core Forecasting:** Nixtla StatsForecast, Nixtla MLForecast
*   **Models:** AutoARIMA, SeasonalNaive, XGBoost Regressor
*   **Feature Engineering:** WindowOps
*   **Visualization:** Plotly Interactive
*   **Data Processing:** Pandas, NumPy

## Интерфейс и работа системы

### 1. Конфигурация и Сегментация
Пользователь задает горизонт планирования и объем выборки. Система автоматически проводит сегментацию портфеля, определяя сложность прогнозирования для каждого SKU.

### 2. Визуализация и Метрики
Интерактивный график позволяет сравнивать факт и прогноз на разных уровнях агрегации. Рассчитываются метрики качества WAPE, MAE, RMSE на валидационной выборке (Holdout). Поддерживается Drill-down от уровня «Все магазины» до конкретного SKU.

### 3. Монитор отклонений (ARM)
Таблица критических алертов. Система подсвечивает товары, требующие внимания планировщика. Реализована логика рекомендаций на основе величины ошибки и типа спроса.

## Дизайн эксперимента

Валидация моделей проводилась методом **Holdout** (отложенная выборка). Последние 28 дней исторических данных исключались из обучения и использовались для проверки.
Для предотвращения утечки данных (Data Leakage) признаки (лаги, скользящие средние) и веса для метода Top-Down рассчитывались строго на обучающей части выборки.

Выбор модели детерминирован сегментом:
*   Стабильный/Эпизодический спрос -> **AutoARIMA** (StatsForecast)
*   Вариативный/Спорадический спрос -> **XGBoost** (MLForecast)
